# 设计规格文档 - 简洁电商购物网站

**风格：** E-commerce Optimized（电商转化优化型）  
**版本：** 1.0  
**创建时间：** 2025-11-11  
**目标用户：** 18-30岁在线购物者  

---

## 1. 设计方向与理念

### 1.1 核心精髓

**转化驱动的极简美学。** 纯白画布让商品成为视觉焦点，大尺寸高清图片占据50-60%视口，黑色或橙色高对比CTA引导一键加购，评分、价格、库存提示等信任元素精准呈现。去除一切广告、弹窗、浮层干扰，让用户专注于"发现-比较-购买"核心流程，在美学克制中实现转化最大化。

### 1.2 真实案例参考

- **Everlane** (everlane.com) - 纯白背景 + 大图展示的极简产品页
- **Warby Parker** (warbyparker.com) - 清爽布局 + 高效筛选系统
- **Muji Online Store** - 克制的视觉语言 + 功能优先
- **Nike/Apple Store** - 黑白高端美学 + 强CTA对比

### 1.3 设计哲学

基于京东电商深度分析的核心洞察：

- **去噪增效**：移除首页轮播、运营浮层、活动H5强曝光
- **决策降本**：筛选主/次级分组、评价维度化、"看相似"链路
- **流程精简**：单页结算、表单自动填充、实时校验
- **信任透明**：价格显性、库存提示克制、物流状态可视、售后入口明确

**权衡说明：** 牺牲品牌故事深度与视觉独特性，换取清晰购买路径与高转化率；适合功能型电商，不适合奢侈品/艺术品平台。

## 4. 布局与响应式策略

### 4.1 页面布局模式

基于 `content-structure-plan.md` 的7大页面类型，定义布局模式：

#### 首页（Home）

**区块序列：**
1. **导航栏（72px固定顶部）** - Sticky Header
2. **Hero区（500-600px）** - 单一主推商品，大图+标题+CTA，无轮播
3. **分类导航（120px）** - 8-12个分类，图标+文字，水平滚动
4. **热门商品区（auto）** - 4列网格（桌面）→ 2列（移动）
5. **新品推荐区（auto）** - 同上网格模式
6. **Footer（auto）** - 帮助、政策、联系方式

**视觉层次：**
- Hero区最大，标题 48-64px
- 区块间距 `48-64px`
- 商品网格间距 `16-24px`

**响应式：**
- 桌面：4列网格（1280px容器）
- 平板：3列网格（768px容器）
- 移动：2列网格（100%宽度，12px间距）

#### 商品列表页（Category/Search）

**布局结构：**
```
[固定导航 72px]
[面包屑 48px]
[筛选侧栏 280px] | [主内容区]
                  | - 排序条（48px）
                  | - 商品网格（3-4列）
                  | - 分页器（64px）
```

**网格策略：**
- 桌面：3列（有侧栏）/ 4列（无侧栏）
- 平板：2列
- 移动：1-2列，筛选改为底部抽屉

#### 商品详情页（Product Detail）

**布局结构（桌面）：**
```
[导航 72px]
[面包屑 48px]
[图片画廊 60%] | [购买盒子 40% - Sticky]
               | - 标题（40px）
               | - 评分+评价数
               | - 价格（32px当前+20px原价）
               | - 库存提示
               | - 规格选择器
               | - 数量选择器
               | - 加入购物车按钮（56px）
               | - 信任标识

[商品详情 Tab 切换] - 全宽
[评价系统] - 全宽
[相关推荐 4列网格] - 全宽
```

**图片画廊：**
- 主图区域 `600×600px`
- 缩略图横向排列，4-6张
- 悬停放大（桌面），捏合缩放（移动）

**Sticky购买盒子：**
```css
.buy-box {
  position: sticky;
  top: 80px;
  max-width: 480px;
}
```

**移动端适配：**
- 图片画廊全宽轮播
- 购买盒子置于图片下方
- 加入购物车按钮固定底部

#### 购物车页（Cart）

**布局结构（桌面）：**
```
[简化导航 + 进度条]
```

## 5. 交互动效与行为规范

### 5.1 动效标准

**允许的动效：**

| 交互 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 按钮悬停 | 背景色变化 + `translateY(-1px)` | 150ms | ease-out |
| 商品卡片悬停 | 阴影加深 + 图片 `scale(1.05)` + 抬升 `-2px` | 250ms | ease-out |
| 快速加购按钮 | 从底部滑入 `translateY(100% → 0)` | 250ms | ease-out |
| 购物车侧栏 | 从右侧滑入 `translateX(100% → 0)` | 400ms | ease-out |
| 模态框遮罩 | 透明度 `0 → 1` | 250ms | ease-out |
| 加入购物车成功 | 按钮缩放 `1 → 1.05 → 1` + 绿色闪烁 | 600ms | ease-in-out |
| 图片放大 | `scale(1 → 1.2)` 鼠标跟随 | 250ms | ease-out |

**禁止的动效：**
- ❌ 自动播放视频/轮播（分散注意力）
- ❌ 视差滚动（影响性能，显得噱头）
- ❌ 过度弹跳/旋转动画（不专业）
- ❌ 宽度/高度/内边距动画（性能差，仅用 `transform` 和 `opacity`）

### 5.2 微交互（提升转化）

**加入购物车成功反馈：**

```css
@keyframes cart-success {
  0% { transform: scale(1); background: #000000; }
  50% { transform: scale(1.05); background: #10b981; } /* 绿色成功 */
  100% { transform: scale(1); background: #000000; }
}
/* 持续 600ms */
```

**库存倒计时（紧迫感）：**
- 库存 ≤5 件时，显示 "仅剩X件" 红色标签
- 库存 = 1 件时，显示 "最后1件！" + 闪烁动画（subtle）

**价格动态更新：**
- 数量变化时，总价淡入淡出更新（200ms）
- 优惠码应用后，折扣价绿色高亮（400ms）

### 5.3 加载状态

| 场景 | 加载样式 |
|------|----------|
| 页面初次加载 | 骨架屏（Skeleton Screen），模拟卡片布局 |
| 商品列表滚动 | 懒加载，底部加载更多按钮 + Spinner |
| 加入购物车 | 按钮内 Spinner（白色），文字变为"加载中..." |
| 结算提交 | 全屏遮罩 + Spinner + "订单提交中..." |

### 5.4 错误状态

| 场景 | 提示样式 |
|------|----------|
| 表单错误 | 红色边框 + 下方红色文字提示 |
| 搜索无结果 | 空状态插图 + "未找到相关商品" + 推荐链接 |
| 库存不足 | Toast提示："库存不足，已调整至最大可购买数量" |
| 支付失败 | 模态框 + 错误信息 + "重试"按钮 |

### 5.5 通知与反馈

**Toast通知：**
- 位置：顶部中央（移动）/ 右上角（桌面）
- 尺寸：最大 `400px` 宽，自适应高度
- 样式：白底，阴影，圆角 `8px`
- 类型：成功（绿色图标）/ 错误（红色）/ 警告（橙色）/ 信息（蓝色）
- 持续：3秒自动消失，可手动关闭

**购物车徽章更新：**
- 加入购物车后，徽章数字跳动动画（scale弹跳）
- 红色背景闪烁一次（200ms）

### 5.6 手势支持（移动端）

| 手势 | 功能 |
|------|------|
| 左右滑动 | 商品图片画廊切换 |
| 捏合缩放 | 商品图片放大查看细节 |
| 下拉刷新 | 商品列表页刷新数据 |
| 上滑 | 隐藏导航栏（给内容更多空间） |

### 5.7 性能规范

**核心Web指标目标：**

| 指标 | 目标值 | 优化策略 |
|------|--------|----------|
| LCP（最大内容绘制） | <2.5s | 商品图片懒加载、WebP格式、CDN加速 |
| FID（首次输入延迟） | <100ms | 减少JavaScript阻塞，事件解耦 |
| CLS（累积布局偏移） | <0.1 | 图片预留空间、骨架屏占位 |

**图片优化：**
- 格式：WebP（备选JPEG）
- 商品卡片：`600×600px` 源图，响应式缩放
- 懒加载：`loading="lazy"`，视口外图片延迟加载
- 占位：低质量图片占位（LQIP）或纯色背景

**动画性能：**
- 仅使用 `transform` 和 `opacity`（GPU加速）
- 避免 `width`、`height`、`margin`、`padding` 动画（触发重排）
- 使用 `will-change` 提示浏览器（谨慎使用）

## 6. 设计原则总结

### 6.1 核心DO's

✅ **视觉层面：**
- 使用纯白背景 `#ffffff`（让商品成为焦点）
- 商品图片占50-60%视口（详情页）
- 高对比CTA（黑色21:1或橙色4.6:1）
- 价格大号粗体（32px红色，原价删除线）
- 最多1个徽章/标签（库存警告，白底黑字）

✅ **布局层面：**
- 3-4列商品网格（桌面），16-24px间距
- 紧凑间距（8pt网格，高信息密度）
- Sticky导航、购买盒子、价格汇总（关键元素始终可见）
- 面包屑导航（明确位置）

✅ **交互层面：**
- 评分+评价数（社会证明）
- 库存提示（紧迫感，克制使用）
- 快速加购（悬停触发）
- 图片缩放（悬停/捏合）
- 加入购物车成功反馈（绿色闪烁）

✅ **流程层面：**
- 单页结算（减少放弃率）
- 表单自动填充（减少输入）
- 实时校验（即时错误反馈）
- 游客结算（无需强制注册）
- 移动支付（Apple Pay、Google Pay）

### 6.2 核心DON'Ts

❌ **禁止的视觉元素：**
- 彩色/渐变背景（干扰商品）
- >3种颜色（视觉混乱）
- 低对比CTA（<4.5:1）
- 多个徽章/标签（图片上最多1个）
- Emoji作为UI图标（使用SVG）

❌ **禁止的布局模式：**
- 首页自动轮播（注意力分散）
- 侧边栏广告（空间浪费）
- 隐藏"加入购物车"（必须首屏可见）
- 小商品图（<50%详情页视口）

❌ **禁止的交互行为：**
- 弹窗广告（任务中断）
- 自动播放视频（烦人、慢）
- 隐藏运费（结算时才显示→放弃）
- 多页结算（增加流失）
- 强制注册（转化障碍）

❌ **禁止的动效：**
- 视差滚动（噱头）
- 宽度/高度动画（性能差）
- 过度弹跳/旋转（不专业）

### 6.3 京东洞察应用

基于 `jd_analysis.md` 的核心发现：

| 京东问题 | 本设计解决方案 |
|----------|----------------|
| 首页运营区块分散注意力 | 移除所有轮播/浮层，Hero区仅1个主推 |
| 列表页广告穿插 | 无广告，商品网格纯净 |
| 筛选选项过多 | 主/次级分组，超过5项折叠 |
| 营销标签噪声 | 徽章最多1个（库存警告），折扣移至价格区 |
| 评价信息过载 | 维度化评分（质量/价格/物流）+ 有用性投票 |
| 结算流程复杂 | 单页结算，自动填充，实时校验 |
| 移动端信息密度低 | 2列网格，紧凑间距，Sticky CTA |
